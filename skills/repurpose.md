# Skill: Content Repurposing

## Purpose
Transform one piece of content into multiple formats across platforms, maximizing reach from a single content investment.

## When to Use
- A blog post is published and needs social distribution
- A case study needs to be turned into sales collateral
- Webinar/video content needs written derivatives
- Any high-performing content should be amplified

## Procedure

1. **Identify the source content** and its key elements:
   - Core insight/thesis
   - Key data points
   - Memorable quotes or statements
   - Actionable takeaways

2. **Map the repurpose plan**:

### From a Blog Post →
| Derivative | Platform | Format | Key Element Used |
|-----------|----------|--------|-----------------|
| LinkedIn post | LinkedIn | Text post | Core insight + data point |
| Twitter thread | X/Twitter | Thread (5-7 tweets) | Step-by-step breakdown |
| Instagram carousel | Instagram | 8-10 slides | Key takeaways as visuals |
| Email highlight | Email | Newsletter section | Summary + link |
| Short-form video script | Instagram/TikTok | 30-60 sec script | One key takeaway |
| Pull quote graphic | All platforms | Image | Best one-liner |

### From a Case Study →
| Derivative | Platform | Format |
|-----------|----------|--------|
| Results spotlight | LinkedIn | Data-focused post |
| Before/after graphic | Instagram | Single image |
| Client quote card | All platforms | Quote graphic |
| Sales one-pager | Internal | Bullet point summary |
| Email proof point | Email nurture | Short case reference |

### From a Webinar/Video →
| Derivative | Platform | Format |
|-----------|----------|--------|
| Blog post recap | Blog | Written summary |
| Key moments clips | Social | 30-60 sec clips |
| Quote cards | Social | Image with text overlay |
| Twitter thread | X/Twitter | Key insights as thread |
| Email to attendees | Email | Follow-up with resources |

3. **Create each derivative** following platform-specific guidelines in `context/editorial.md`

4. **Deliver as a repurpose package**:
```
# Repurpose Package: [Source Content Title]

Source: [link/reference]
Date: [original publish date]
Pillar: [pillar]

## Derivatives Created:
1. [Platform] — [Format] — [Status]
2. [Platform] — [Format] — [Status]
...

## Posting Schedule:
- Day 1: [derivative]
- Day 3: [derivative]
- Day 5: [derivative]
...
```

## Rule
Never copy-paste the same content across platforms. Each derivative must be **native to its platform** — different hook, different format, different length. Same core message, different execution.
