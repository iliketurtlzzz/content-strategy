# Skill: Blog Post Creation

## Purpose
Write a complete, publish-ready blog post for MarketWake.

## When to Use
- An outline has been approved (use `article-outline` skill first for complex topics)
- User requests a full blog post
- Content calendar calls for a new post

## Procedure

1. **Confirm prerequisites**:
   - Target keyword identified
   - Persona selected
   - Pillar assigned
   - Outline approved (or topic simple enough to skip)

2. **Write the post following editorial standards** (see `context/editorial.md`):
   - Title: Keyword-optimized, benefit-driven or curiosity-driven
   - Meta description: 150-160 characters
   - Introduction: Hook + promise, keyword in first 100 words
   - Body: H2/H3 structure, 200-300 words per section
   - Paragraphs: 2-4 sentences max
   - Include data points, examples, or MW client results
   - Bold key takeaways
   - Internal links: 2-3 minimum to MW pages

3. **Apply brand voice** (see `context/brand.md`):
   - Confident, direct, smart
   - No filler, no jargon soup
   - Lead with insight, not introduction
   - Active voice throughout

4. **Close strong**:
   - Key takeaway summary
   - Clear CTA tied to MarketWake service
   - Suggested related posts for internal linking

5. **Deliver with metadata**:
   ```
   ---
   Title:
   Meta Description:
   Primary Keyword:
   Secondary Keywords:
   Pillar:
   Target Persona:
   Word Count:
   Suggested URL Slug:
   Internal Links:
   CTA:
   ---
   ```

## Word Count Guidelines
- Standard blog post: 1,000-1,500 words
- Deep-dive guide: 2,000-3,000 words
- Quick tip/news post: 500-800 words
